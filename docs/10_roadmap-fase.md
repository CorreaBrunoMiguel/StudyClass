# Roadmap por Gates (v1)

## Gate 1 — Requisitos e Regras (FECHADO)

- Regras de progressão (unidade/módulo/curso)
- Fórmula de notas
- Whitelist de e-mails
- Conteúdo em JSON por blocos

## Gate 2 — Especificação

- Modelo de dados (entidades + relações)
- Fluxos principais (aluno e admin)
- APIs (contratos)
- Decisão de armazenamento do conteúdo JSON (DB vs arquivos)
- Regra de tentativas (melhor vs última)

## Gate 3 — Arquitetura

- Stack (frontend, backend, banco)
- Auth (session/jwt) + Google OAuth
- Deploy alvo + pipeline CI/CD
- Observabilidade mínima

## Gate 4 — Implementação

- Épicos por módulo do sistema

## Gate 5 — Testes

- Unit + integration + e2e mínimo

## Gate 6 — Deploy Beta

- Ambiente + variáveis + smoke tests
